⏺ 我看到了——REST join_game 是原子操作，它同时设置第二个玩家ID 和 将status改为
  playing。所以流程是：

  1. A创建游戏 → DB: status=waiting, red_player_id=A, black_player_id=NULL
  2. A连WS → was_waiting=true, both_players_present=false → 不激活 ✓
  3. B调用REST join_game → DB: status=playing, black_player_id=B (原子操作)
  4. B连WS → was_waiting=false (status已是playing) → 不激活 ❌

  问题变成了：第二个玩家连WS时，status已经是
  playing，was_waiting=false，所以不会激活。

  正确的判断应该是：当游戏状态是 playing 
  且时间控制尚未激活时，激活它。或者更简单：当两个玩家都在房间里时才激活。

     Game Replay Feature

     Context

     Users cannot review finished games. The server already stores full move
     history (MoveEntry with FEN, color, check, time per move) and has a GET 
     /api/games/{id}/moves endpoint, but the frontend has no replay UI. The
     ChessBoard.vue component already renders any FEN string, making it directly
     reusable for replay.

     Implementation

     Phase 1: Server — Add initial_fen to GameInfo

     initial_fen is needed to show the board before any moves (step 0).
     Currently only stored in DB, not returned by API.

     - chess-server/src/db/models.rs: Add pub initial_fen: Option<String> to
     GameInfo struct
     - chess-server/src/handlers/game_handler.rs: Add initial_fen: 
     game.initial_fen in build_game_info()
     - chess-server/src/services/game_service.rs: Add initial_fen:

⏺ 非常方便，这正是 Docker 的核心优势。流程如下：

  修复→更新流程

  Mac 上修 bug → docker compose build → 推送镜像 → Ubuntu 拉取 → 重启容器

  具体有两种方式：

  方式 1：Docker Hub / 私有镜像仓库（推荐远程服务器）

  # Mac 上
  docker compose build
  docker tag chinese-chess-server:latest yourname/chinese-chess-server:latest
  docker tag chinese-chess-client:latest yourname/chinese-chess-client:latest
  docker push yourname/chinese-chess-server:latest
  docker push yourname/chinese-chess-client:latest

  # Ubuntu 上
  docker pull yourname/chinese-chess-server:latest
  docker pull yourname/chinese-chess-client:latest
  docker compose up -d   # 自动用新镜像重启

  方式 2：直接 scp 镜像文件（无镜像仓库时）

  # Mac 上
  docker compose build
  docker save chinese-chess-server chinese-chess-client | gzip > images.tar.gz
  scp images.tar.gz user@ubuntu-server:~/

  # Ubuntu 上
  docker load < images.tar.gz
  docker compose up -d

  方式 3：Ubuntu 上直接 build（最简单，但需要源码）

  # Mac 上：git push
  # Ubuntu 上：
  git pull
  docker compose build
  docker compose up -d

  关键点

  ┌────────────────────┬────────────────────────────────────────────────────┐
  │        问题        │                        答案                        │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ 数据会丢吗？       │ 不会。PostgreSQL 数据在 Docker volume 里，up -d    │
  │                    │ 不影响数据                                         │
  ├────────────────────┼────────────────────────────────────────────────────┤
  │ 需要停机吗？       │ 几秒钟。up -d 自动滚动重启，数据库不重启           │
  ├────────────────────┼────────────────────────────────────────────────────┤
